---
description: How to evaluate edits involving spatial positioning, counting, and object
  relationships.
name: spatial-and-relational-reasoning
type: skill
---

When an instruction specifies a spatial relationship (e.g., "under the bag on the right", "in front of the catcher") or counting (e.g., "4th surfboard"):

1. **Establish the Anchor**: Identify the reference objects first (e.g., "the bag on the right", "the catcher").
2. **Verify the Target**: Check if the target object (e.g., "cat's foot", "home plate") is present.
3. **Check the Relationship**: Confirm the exact spatial relationship. "Under" means physically beneath, "in front of" means between the camera and the anchor object.
4. **Counting**: Always count systematically (e.g., left-to-right or top-to-bottom). Verify the total number of items to ensure you are identifying the correct "Nth" item.
5. **Common Failure Modes**:
   - The edit is applied to the wrong object (e.g., the 3rd surfboard instead of the 4th).
   - The spatial relationship is inverted (e.g., "above" instead of "under").
   - The target object is added, but the anchor object is accidentally removed or distorted.
